# Report Quality Standard

Use this file as a final gate before finishing any report.

## Minimum Source Mix

Try to cover at least these source classes for each asset:

1. Official source:
   docs, blog, roadmap, governance forum, GitHub, official X; for stocks also include IR page, earnings release, shareholder letter, annual or quarterly report
2. Market or on-chain source:
   explorer, analytics dashboard, token page, staking or treasury data; for stocks also include quote page, valuation dashboard, consensus or market-data source
3. Social discussion source:
   direct X posts, founder posts, researcher posts, or reliable summaries
4. Secondary research source:
   exchange research, analytics firm, major publication, or reputable report

For unlock-heavy assets, add a fifth source class when possible:

5. Unlock / vesting source:
   official tokenomics page, vesting schedule, governance post, treasury disclosure, vesting contract, or a reputable unlock dashboard as fallback

For stock assets, add a sixth source class when possible:

6. Filing / earnings source:
   10-K, 10-Q, 20-F, 8-K, earnings deck, call transcript, investor day, or exchange filing

For chain or network assets, add a seventh source class when possible:

7. Node / validator source:
   official validator page, explorer, staking dashboard, telemetry site, client dashboard, or reputable infrastructure tracker

If one class is unavailable, say so in the report instead of pretending coverage exists.

## Freshness Standard

- Prefer sources from the last 30-90 days for roadmap, ecosystem, and X discussion.
- For fast-moving narratives, prefer the most recent few weeks.
- On repeat coverage, explicitly prioritize what happened after the date of the latest prior report.

## Must-Have Analytical Moves

- State what changed.
- State why it matters economically.
- Include `历史大事件与影响意义` and make sure the chosen events changed the trajectory of the asset, not just generated headlines.
- State the recent and upcoming unlock profile, or say explicitly that the asset is largely fully circulating or lacks reliable unlock disclosures.
- For chain or network assets, state the node count and/or validator count, clarify what class of nodes the number refers to, and explain whether decentralization is improving, stable, or weakening.
- For stocks, state the key equity metrics: revenue, growth, margins, EPS or free cash flow, net cash or debt, diluted share count, and the main valuation multiples the market is using.
- State who benefits and who gets diluted, displaced, or weakened.
- State whether the token or equity holder actually captures value from the activity described.
- State what would falsify the current thesis.
- Choose a primary sector and make sure the metrics and comparables match that sector.
- In `全方位多角度分析`, cover at least 8 relevant angles or explicitly mark some angles as not applicable.
- Include a real `bull case`, a real `bear case`, and at least one `market may be missing` variable.
- If the request is chain-centric, separate `链是否在变强` from `原生代币是否受益`.
- If the request is chain + AI fit, rank at least 2-3 AI paths and explain why the top path wins.
- For price analysis, make the default horizon the next 12 months unless the user explicitly asks for a different time window.
- Include 2-4 close comparables when a reasonable peer set exists, and explain whether this asset deserves a premium, parity, or discount versus peers.
- In `股市的影响`, name the most relevant equity benchmarks, sectors, or listed-company proxies rather than referring vaguely to "the stock market."
- In `国际战争的影响`, name the most relevant geopolitical channels such as sanctions, energy, safe-haven flows, cyber risk, or regional exposure rather than referring vaguely to "war risk."

## Avoid

- Empty adjectives such as "strong", "good", "huge", or "promising" without evidence.
- Repeating roadmap items without saying whether they shipped, slipped, or remain vague.
- Confusing ecosystem or business growth with holder value capture.
- Treating AI narrative as AI utility without proof.
- Treating price targets as certainty.
- Writing only from the team's perspective without testing the skeptical case.
- Calling a chain `AI-native` without showing what concrete AI workload or product should live there.
- Choosing comparables that are famous but not actually comparable.
- Ignoring unlock pressure for assets with material non-circulating supply.
- Claiming stock-market influence without explaining which stocks, sectors, or liquidity channels matter.
- Claiming war impact without explaining the exact geopolitical transmission path.
- Claiming decentralization or validator strength without showing node or validator evidence when such data should exist.
- Using tokenomics language on equities without translating it into share count, dilution, buybacks, margins, and cash flow.
- Using formulaic contrast rhetoric built around negation instead of writing the conclusion directly and then supporting it with evidence.

## Useful Optional Additions

- `关键指标快照`: market cap, FDV, circulating supply, fully diluted supply, staking ratio, fees, TVL, active addresses, or throughput when relevant
- `解锁与供给压力快照`: especially useful when investor, team, foundation, or ecosystem unlocks matter
- `节点与验证者概况`: especially useful for L1/L2 or network-centric reports where security and decentralization matter
- `股票信息与核心指标`: especially useful when the asset is a listed company or the user asks for a stock
- `历史大事件与影响意义`: especially useful when the user wants deeper context on how the current thesis was formed
- `与上次相比的关键变化`: especially useful for repeat coverage
- `未证实但值得跟踪的问题`: useful when the market is debating unresolved items
- `多角度覆盖矩阵`: useful when you want a one-screen view of strengths, weaknesses, and non-applicable angles
- `核心投资逻辑 / 反方观点 / 市场忽略变量`: useful when the user wants the report to be especially deep
- `对标项目比较`: useful when the user wants relative positioning, valuation context, or same-category comparison
