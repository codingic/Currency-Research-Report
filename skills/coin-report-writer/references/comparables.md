# Comparables

Use this file whenever the report should compare the asset with similar chains or same-category projects.

## Core Rule

Do not compare against random famous projects just because they are large.

Choose comparables that are actually close in:

- sector
- product category
- user problem
- technical architecture
- token design
- target customer

## Selection Rule

- Prefer 2-4 comparables.
- Choose the closest same-sector peers first.
- For chains, compare with similar chains or rollups.
- For DeFi, compare with the same protocol category.
- For AI, compare with the same AI subcategory, not generic AI coins.
- For stocks, compare with companies that share the same business model, customer base, margin structure, and valuation framework.
- If the project is unusual, explain why comparable sets are imperfect.

## Good Comparable Examples

- L1/L2 chain:
  compare with similar-performance or similar-positioning chains/rollups
- DEX:
  compare with other DEXs, not with lending protocols
- Oracle / middleware:
  compare with similar infra and data-layer projects
- AI compute:
  compare with compute, data, or inference networks, not meme AI tokens
- Meme:
  compare with meme tokens that compete for the same attention/liquidity pool
- Stock / listed company:
  compare with the closest operating peers, not just the biggest or most famous stocks in the market

## Comparison Dimensions

Use the most relevant set, but normally include:

1. 定位
   what problem each project is trying to solve
2. 技术与架构
   performance, decentralization, interoperability, design tradeoffs
3. 用户与生态
   developers, apps, users, distribution, integrations
4. 经济模型
   emissions, unlocks, fees, value capture, treasury, token utility; for stocks use revenue model, margin profile, dilution, buybacks, and balance-sheet structure
5. 市场结构
   liquidity, listings, derivatives, holder structure, valuation premium or discount; for stocks also include index flows, factor exposure, and institutional ownership where relevant
6. 护城河与风险
   switching costs, brand, network effects, regulation, execution risk

## Output Pattern

When comparables materially matter, include:

1. `对标项目比较`
   name the peer set and why they are the right comparison group
2. `相对优势`
   where this asset is better
3. `相对劣势`
   where this asset is weaker
4. `市场给的估值是否合理`
   whether the asset trades at a justified premium, discount, or mismatch

## Common Mistakes

- comparing a chain to BTC just because BTC is large
- comparing across unrelated categories
- ignoring that some comps have stronger token capture even if weaker product growth
- listing peers without reaching a judgment
