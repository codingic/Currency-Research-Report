# Currency Research Report

一个面向 `Codex / AI 研究工作流` 的研究仓库，包含两部分内容：

- `reports/`：已经生成好的币种、链、股票深度研究报告
- `skills/coin-report-writer/`：用于自动生成研究报告的 Codex skill

这套 skill 默认输出中文报告，强调 `全面`、`深刻`、`可追踪更新`，适合做加密资产、链、项目、股票的研究归档。

## 功能说明

这套仓库当前提供两类核心功能：

- `研究结果归档`
  把已经完成的币种、链、股票研究报告按日期保存到 `reports/`，便于后续检索、对比和持续更新。

- `研究报告自动生成`
  通过 `skills/coin-report-writer/` 里的 Codex skill，把一个简单输入例如 `BTC`、`SOL`、`AAPL`、`pump` 扩展成完整的深度研究文档。

这个 skill 的工作目标不是只写“项目更新”，而是输出接近 buy-side / research desk 风格的完整分析，重点覆盖：

- 标的定位与核心投资逻辑
- 反方观点、证伪条件、市场误判点
- 代币经济模型或股票资本结构
- 节点 / 验证者、解锁、估值、竞争、治理、安全
- 宏观、监管、股市风格、国际战争等外部变量
- 与 AI 的相关性，以及链与 AI 的最佳结合路径

## 工作方式

仓库里的 skill 采用的是 `一个输入 -> 一份独立报告` 的工作流：

- 输入一个币种、链名、股票代码，默认生成一份完整报告
- 输入多个标的时，分别生成多份文件，而不是合并成一份
- 如果同一标的已经写过旧版本，会优先按照“更新版”逻辑处理，先写新变化，再补最新判断
- 默认会先抓取最新资料，再整理成固定结构的中文研究文档

## 写作约定

为了让输出长期可读、可比较，当前仓库默认采用这些约定：

- 默认输出 `简体中文`
- 默认 `一个标的一份文档`
- 默认写 `深度版`，不是短评版
- `关键指标快照` 只写结论和口径，不在这一节单独写来源
- 更看重 `链前景` 和 `代币前景` 的区分，而不是把两者混为一谈
- 如果是股票，会用股票研究框架而不是强行套加密模板
- 如果是更新版，会优先写自上次以来真正改变判断的变量

## 输出效果

生成后的报告默认是“可长期追踪”的结构化文档，适合用于：

- 建立单个标的的长期研究档案
- 做同赛道横向比较
- 追踪同一标的在不同时间点的判断变化
- 给后续自动化、归档、审阅或二次加工提供稳定输入

## 报告特点

- 默认 `一个币种 / 链 / 股票 = 一个单独文档`
- 默认输出 `简体中文`
- 同一标的再次生成时，会优先写 `新发生的事情`
- 支持 `加密资产`、`链`、`股票`
- 强制加入 `对标项目比较`
- 强制加入 `解锁信息`、`节点/验证者信息`
- 强制加入 `宏观经济`、`经济政策`、`股市影响`、`国际战争影响`
- 强制加入 `历史大事件与影响意义`
- 支持 `未来 12 个月价格走势预测`
- 支持 `链与 AI 最佳结合方式` 分析

## 如何安装 Skill

把 skill 复制到本机 `Codex` 的 skills 目录：

```bash
cp -R skills/coin-report-writer ~/.codex/skills/
```

安装后，新会话里直接输入币种、链名或股票代码即可触发。

## 如何使用

你可以直接输入：

```text
BTC
ETH
SOL
ICP
NVDA
分析此链未来的发展前景
分析此链与 AI 最好的结合方式
预测一年内价格走势
```

也可以一次输入多个标的：

```text
BTC ETH SOL
```

系统会分别生成多份独立文档，而不是合并成一份。

## 文档输出位置

默认输出到工作区的：

```text
reports/
```

文件名格式类似：

```text
2026-03-19-btc-analysis.md
2026-03-19-solana-analysis.md
2026-03-19-nvda-analysis.md
```
