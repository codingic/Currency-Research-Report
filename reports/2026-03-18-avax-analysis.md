## 与上次相比的关键变化

本次为首次覆盖，不存在内部旧版对照。如果只看最近 3-6 个月，Avalanche 最重要的新变化有六个：

- `2026-03-06`，`Retro9000 C-Chain Round 1` 正式上线，开始按真实链上活动和 `AVAX burned through gas fees` 来奖励建设者。这很关键，因为 Avalanche 终于在公开叙事里把“生态增长”和“AVAX 销毁”更直接地绑在一起。

- 市场对 Avalanche 的看法也变了。过去更常把它当成“Subnets/Gaming 链”，现在更该把它看成 `可定制 L1 平台 + 跨链通信层 + 机构与 RWA 友好型基础设施`。

## 赛道定位

- 主赛道：L1 公链
- 次赛道：可定制 L1 / Appchain 基础设施 / 机构与 RWA 友好型链

Avalanche 不能只用“普通高性能公链”框架去看。

更准确的理解是：

- 它既有一条主网经济核心，也有一整套允许项目方发起专用 L1 的平台；
- 它并非单纯押注一条全局状态链的规模，更准确地说，是押注“许多垂直链和定制化网络会在 Avalanche 体系里出现”；
- 它最有差异化的方向，并非 meme 或单一 DeFi 热度，更关键的是 `定制化 L1 + 原生互操作 + 合规/机构场景 + 低成本部署`。

所以判断 Avalanche，要拆成两个问题：

1. Avalanche 这套多链平台未来会不会变得更重要；
2. `$AVAX` 能不能成为这个平台的真实价值汇聚层，而非只做启动燃料。

## 核心投资逻辑

AVAX 最强的多头逻辑，并非“便宜版 SOL”；核心在于：

1. Avalanche 现在更像一个 `可定制区块链操作系统`，而不只是单链 L1。
2. Etna / Avalanche9000 把部署 L1 的成本和复杂度显著压低，让此前偏理想化的 Subnet 叙事重新具备商业可行性。
3. Avalanche 的差异化并非空泛“企业合作”，更准确地说，是开始出现更贴近真实资本市场的用例：`Progmat`、`FUSD`、日本代币化证券、亚洲机构流动性等。
4. AVAX 的价值捕获路径比很多公链更复杂但也更多元：Primary Network 质押、C-Chain 费用销毁、Avalanche L1 验证槽位费用销毁、以及定制 L1 可能带来的桥接和流动性回流。

一句话概括：

AVAX 的核心投资逻辑，是 Avalanche 有机会成为 `合规友好型多链金融与垂直应用平台`，而非只靠 C-Chain 自己卷 TPS 和 TVL。

## 反方观点与证伪条件

最强的反方逻辑同样很成立：

1. Avalanche 的“平台化”叙事其实并非新东西，Subnets 早就讲过，市场有理由怀疑这只是一次重新包装。
2. 即使更多 Avalanche L1 启动，价值也可能被各自 L1 的原生代币、国库或应用层截留，而非回到 AVAX。
3. Avalanche 当前的链上收入很低。DefiLlama 页面显示，24 小时链上 revenue 只有几千美元量级，这说明“活跃度”并没有自然转化成“强捕获”。
4. 与 Solana 相比，Avalanche 仍然缺少那种让市场一眼看懂的单链爆发性；与 Cosmos 相比，它的多链叙事又更复杂、更依赖核心团队推动。

关键证伪条件：

- 如果未来 6-12 个月，Retro9000、定制 L1、RWA、稳定币继续发布，但 `AVAX 销毁、质押需求、链上收入、稳定币净流入` 都没有明显改善，那么“平台增长会回流给 AVAX”这条主逻辑就会被明显削弱。
- 如果 Progmat、FUSD 这类机构案例长期停留在示范级别，而没有形成可持续的链上活动和资产沉淀，Avalanche 的机构溢价会被市场打折。
- 如果新启动的 Avalanche L1 大多不带来 C-Chain 流动性、ICM 使用或 AVAX 需求，AVAX 可能继续面临“平台在扩张，但代币未必受益”的问题。

## 市场可能忽略的变量

- 市场可能低估的变量一：`1.33 AVAX/月` 的 L1 验证槽位费用被销毁，这让 Avalanche 相比纯 appchain 生态多了一道更明确的 AVAX 价值回流闸门。
- 市场可能低估的变量二：Avalanche 的机构进展，最先反映的未必是 DeFi TVL，而更可能先体现在 `稳定币、证券代币化、权限型或半权限型 L1`。
- 市场可能低估的变量三：Etna 把 C-Chain 最低 base fee 从 `25 nAVAX` 降到 `1 nAVAX`，短期会让单笔费收更低，但长期可能让 Avalanche 更适合高频应用和更密集的交互型业务。
- 市场可能低估的变量四：Avalanche 的真正竞争对象不只是 Solana 和 Sui，也包括 Cosmos/Polkadot 这类“让别人来发链”的生态；而 AVAX 是否比这些项目更能捕获平台价值，是中期关键。
- 市场可能低估的变量五：Avalanche 若在日本和亚洲的合规金融基础设施里站稳，它得到的可能并非纯 crypto beta 溢价，更准确地说，是更偏“数字金融底层软件”估值框架。

## 对标项目比较

Avalanche 的最佳对标，不该只看“同是 L1”，而要看谁也在争夺 `高性能执行层`、`应用专链`、`跨链平台` 或 `合规金融基础设施`。我这里选：

- 对标对象：Solana、Sui、Cosmos Hub
- 为什么选它们做对比：
  `Solana` 代表单链高性能执行和强零售/资本市场心智；
  `Sui` 代表新一代高性能 L1 与更清晰的新资产叙事；
  `Cosmos Hub` 代表 appchain / sovereign chain 的平台化老路线。

- 相对优势：
  Avalanche 相对 Solana、Sui 的优势，是 `可定制 L1 + 原生互操作 + EVM 兼容 + 对机构和权限场景更友好`；相对 Cosmos 的优势，是对 EVM 开发者更友好、Primary Network 与 AVAX 的价值锚更清晰一些。
- 相对劣势：
  相比 Solana，Avalanche 缺少单链世界的爆发式网络效应；相比 Sui，叙事不够新，零售情绪也弱；相比 Cosmos，Avalanche 的多链设计更依赖 AVAX 与主网经济，但这也让用户更难快速看懂。
- 当前估值相对同类是溢价、平价还是折价：
  我认为 AVAX 当前对 Solana 和 Sui 明显处于折价定价，这有合理性，因为市场热度和单链活动不如它们；但如果站在“可定制 L1 平台 + 机构/RWA”框架下看，AVAX 对纯粹 appchain 叙事又不算贵，核心在于其价值捕获是否能兑现。

结论：

- 如果你相信未来是“单一大链吃掉一切”，AVAX 不一定是最优解；
- 如果你相信未来会是“多条垂直链 + 合规场景 + 跨链消息互通”，Avalanche 的估值可能存在被低估的部分。

## 执行摘要

Avalanche 现在最重要的变化，并非单一性能升级，更准确地说，是它的商业叙事正在从“高性能 EVM 公链”转向“可定制 L1 平台 + 机构与 RWA 友好型基础设施”。过去几个月，增量非常集中：`Retro9000` 开始用真实链上活动和 AVAX 销毁来激励建设者；`Progmat` 把 `20 亿美元以上` 的代币化证券迁移到 Avalanche；`FUSD` 把亚洲收益型 RWA 稳定币带到 Avalanche；而 Etna 带来的 `99.9%` L1 成本下降、`1 nAVAX` 的 C-Chain base fee 和标准化 ICM，仍在继续重塑平台经济。链本身的前景我判断为 `中强偏强`，因为它在多链平台、机构友好和定制化部署上仍有清晰差异化；但代币前景我只给 `中性偏强`，因为 AVAX 的最大问题并非“有没有故事”，更接近“平台增长能否真正回流成 AVAX 销毁、质押需求和流动性锚”。未来一年，AVAX 最关键的验证点有三件事：`新 L1 是否持续上线并活跃`、`RWA/稳定币是否形成真实沉淀`、`AVAX 的 burn 与 staking 需求是否更清晰地抬升`。如果三者共振，AVAX 有机会从“过气 L1”重新被定价为有制度优势的平台资产；如果不共振，它仍可能继续停留在“架构很好、但币价捕获一般”的状态。

- `2026-02-27`，官方宣布 `Retro9000` 第四批 L1 与基础设施工具获选项目，说明 Avalanche 仍在把生态扶持重点放到“发链、工具、底层能力”上，而非单纯补贴前端应用。
- `2026-02-25`，Progmat 把 `20 亿美元以上` 的代币化证券迁移到 Avalanche，并选择专用 Avalanche L1，重点强调链上隐私与定制能力。
- `2026-02-09`，Fosun Wealth 把 `FUSD` 放到 Avalanche，突出亚洲机构资产与全球流动性的连接。
- `2026-01-21`，Avalanche Foundation 重新强调 `Retro9000` 是一项总规模 `4000 万美元` 的项目，围绕 L1 和开发者工具，按真实交易带来的 AVAX 销毁效果分发激励。
- 官方首页和 Builder Hub 当前都在强化同一套方向：`Powered by Avalanche`、`purpose-built L1s`、`Interchain Messaging`、`builder console`、`InfraBUIDL(AI)`、`Institutions and Capital Markets`。

这些变化为什么重要：

- Avalanche 正在从“公链生态竞争”切到“平台生态竞争”；
- 机构和 RWA 叙事不再只是 logo 展示，更准确地说，是开始出现有规模的资产迁移；
- 但 Avalanche 也因此更依赖长期平台化执行，而非短期交易热度。

## 2. 最近 X 上的讨论

最近 X 上围绕 AVAX 的讨论，主线相当集中：

- 第一条：机构与 RWA。`2026-02-25`，官方 `@avax` 直接发帖强调“Another $2B+ of RWAs is headed to Avalanche”，把 Progmat 和日本数字资产基础设施放到台前。这说明官方希望市场重新把 Avalanche 与 `tokenization`、`privacy`、`institutional rails` 联系起来，而非只看 DeFi。
- 第二条：Retro9000 和 builder economy。市场开始讨论 `Retro9000` 的意义，不再只是一个 grant 名字，更接近“是否终于出现把 builder 激励和 AVAX burn 绑定起来的机制”。
- 第三条：Avalanche Summit `2026-09-16` 至 `2026-09-17` 在纽约举办，社区普遍把它视为接下来一轮生态发布和市场预期重建的重要时间点。
- 第四条：多头更愿意强调 Avalanche 的“平台股”属性，即 `L1 launchpad + institutional settlement + custom network stack`。

我的判断：

- X 上对 Avalanche 的主流共识，已经从“Subnets 会不会火”升级成“这套定制链平台到底能不能产生可见的 AVAX 需求”；
- 多头看机构、L1 平台和合规资产；
- 空头看收入弱、估值锚弱、代币回流机制仍不够锋利。

## 3. 经济模型

AVAX 的经济模型，核心是“有上限的质押型平台资产”，但进入 L1 时代后，其捕获路径比以前更分散。

- 供应端：
  当前流通约 `4.3177 亿`，总供应约 `4.6344 亿`，最大供应约 `7.1575 亿`。
- 质押与发行：
  验证者最低质押 `2000 AVAX`，委托最低 `25 AVAX`，质押周期 `2 周到 1 年`。奖励由一个有最大供给约束的公式决定，而非无限增发。
- Primary Network 安全：
  Avalanche 仍然要求 Primary Network 质押，这是 AVAX 最基础的需求来源。
- 手续费与销毁：
  DefiLlama 的 `Fees / Revenue` 页面明确把 Avalanche 的链上 revenue 定义为被烧毁的 AVAX 交易费，这意味着 Primary Network 使用增长可以转化为供给收缩。
- L1 平台化后的新变量：
  文档显示，Avalanche L1 验证者需要支付约 `1.33 AVAX/月` 的验证槽位费用，且这笔费用被烧毁；同时，Avalanche L1 还可以通过 `Reward Manager` 自定义本链手续费分配，是给验证者、国库还是烧毁。

这套模型的优点：

- AVAX 并非纯治理币，更准确地说，是安全、费用、平台准入和部分销毁的复合资产；
- 最大供应上限使它天然比无上限增发模型更容易讲长期稀缺性；
- L1 验证槽位费用，让 Avalanche 平台扩张至少保留了一层对 AVAX 的需求锚。

但问题同样明显：

- 当前链上 revenue 仍然很小，说明真实 burn 规模不足；
- C-Chain base fee 降到 `1 nAVAX` 对采用是利好，对单位交易收入却是利空；
- 更致命的是，Avalanche L1 允许很高的经济自主性，平台越成功，价值越可能分散在各条 L1 的本地经济里。

所以 AVAX tokenomics 的结论是：

- 方向比很多老牌 L1 更先进；
- 但强度并不算非常硬；
- 真正的重估，需要平台活跃度、AVAX burn 和质押需求一起改善。

## 4. 前景分析

如果只看链的发展前景，我对 Avalanche 未来 3-5 年的判断是 `中强偏强`。

原因有五个：

- 第一，Avalanche 对“未来不止一条链”的判断并没有过时，反而因为机构、游戏、合规资产、企业工作流的差异化需求而更有现实意义。
- 第二，Etna 把此前 Subnet 叙事中最痛的经济和部署门槛大幅降低，这对平台型扩张非常关键。
- 第三，Avalanche 的原生互操作工具链比很多竞争对手更清晰，`ICM / Warp Messaging / Interchain SDK` 让跨 L1 通信并非附加项，更准确地说，是产品主线。
- 第四，它对机构和合规场景更友好。Progmat、FUSD 这类案例，不一定最性感，但更可能形成持久的链上资产沉淀。
- 第五，Avalanche 在游戏、企业和资本市场之间的兼容性仍然强，这让它不像某些链那样只能押一条叙事。

但如果看代币前景，就必须拆开：

- 链的前景：`中强偏强`
- 币的前景：`中性偏强`

原因在于：

- 平台扩张不自动等于代币捕获扩张；
- Avalanche 越支持主权化 L1，越要回答“到底哪些价值一定要经过 AVAX”。

所以我对 Avalanche 的核心判断是：

- 作为链，它仍有成为“多链世界基础设施供应商”的机会；
- 作为币，AVAX 需要更明确地证明自己并非可替换的启动燃料，更准确地说，是平台增长的必要受益者。

## 5. 捕获价值分析

AVAX 的价值捕获路径，主要有五条：

1. Primary Network 质押需求
   所有 Primary Network 验证者都需要质押 AVAX，这是最底层的安全锚。
2. C-Chain 与主网手续费销毁
   链上交易越多，理论上烧毁的 AVAX 越多。
3. Avalanche L1 验证槽位费用
   文档显示，每个 Avalanche L1 的验证槽位费用会向 P-Chain 支付并销毁，这是平台化后最值得重视的新捕获路径之一。
4. 平台带来的流动性回流
   如果定制 L1 仍需要通过 C-Chain、桥、DEX 或钱包生态完成流动性和结算，AVAX 可能作为中心流动性资产间接受益。
5. 市场预期与制度溢价
   当市场相信 Avalanche 能承接更多 RWA、稳定币和专用链时，AVAX 会获得平台资产溢价。

但限制必须写清楚：

- Avalanche L1 可以高度自定义费用分配，这意味着很多经济价值可能停留在各自链内；
- 当前 DefiLlama 显示的 `App Revenue` 明显高于链本身 revenue，说明生态热度和原生代币捕获之间仍有缝隙；
- AVAX 的价值捕获逻辑，远没有 BNB 那样直观，也没有 BTC 那样单一清晰。

我的结论：

- AVAX 的捕获路径并不少；
- 但真正决定其估值的，是哪几条路径能从“理论可行”变成“季度级别可验证”；
- 在那之前，AVAX 更像“有选择权的平台资产”，而非已经证明高现金流回流的资产。

## 9. 与 AI 相关的重点分析

如果问 Avalanche 与 AI 最好的结合方式，我的答案并非“去做 GPU 市场”；核心在于：

- 最佳结合方向：
  `面向企业、游戏和垂直场景的 AI agent 专用 L1 与结算层`
- 为什么最适合这条链：
  Avalanche 的优势不在通用 AI 计算，而在 `可定制 L1`、`权限/合规可配置`、`EVM 兼容` 和 `跨 L1 原生消息通信`。这很适合那些需要 AI agent 自动执行、资产交互、权限控制和审计留痕的场景。
- 次优方向：
  `AI 驱动的游戏经济`、`AI agent 支付与金库管理`、`机构内部自动化工作流`
- 不适合的 AI 路径：
  去中心化 GPU/算力市场、链上大模型训练、通用 AI 计算层。这些都并非 Avalanche 当前最强的产品契合点。
- 落地路径：
  在 Avalanche 上为特定 agent 场景发起专用 L1，通过 ICM/Teleporter 与其他 Avalanche L1 或 C-Chain 连接，资金和 DeFi 流动性仍回到 C-Chain 或主流桥接入口。
- 价值捕获：
  如果 AI 场景通过 Avalanche L1 大规模落地，AVAX 可以从验证槽位费用、Primary Network 质押和部分跨链/主网交互中受益；
  但若价值主要停留在专用 L1 的本地 token 或应用收费中，AVAX 的受益会偏间接。

结论：

Avalanche 与 AI 最好的关系，并非“AI 公链”，更接近“让特定 AI agent 网络拥有自己链、又能互通和结算的基础设施平台”。

## 10. 最新路线图

结合 Avalanche 官网、Builder Hub 和最近几个月的公开发布，我认为 Avalanche 当前路线图可以概括为五条主线：

- `Etna / Avalanche9000` 后续效应释放
  继续围绕更便宜的 L1 启动、更低 C-Chain base fee、更强主权性和更可扩展的多链结构展开。
- `Interchain Messaging`
  通过 ICM、Warp Messaging、Interchain SDK，把多条 Avalanche L1 的互通变成产品默认能力。
- `Retro9000`
  继续用结果导向的方式激励 L1、工具、开发框架和真实活动。
- `Institutions and Capital Markets`
  把 Avalanche 打造成更适合稳定币、代币化证券、真实资产和合规型结算网络的基础设施。
- `Builder Tooling`
  Builder Console、CLI、文档、Snowtrace 验证、合约部署和 L1 管理工具继续完善。

这份路线图的优点：

- 方向很清晰，并非毫无重点地撒网；
- 技术、平台和商业化之间相互支撑。

缺点也很清楚：

- 每条线都需要时间；
- 真正让 AVAX 重估的，并非“又多一个路线图词汇”，更准确地说，是平台收入和销毁的可见提升。

## 11. 宏观经济对此代币的影响

AVAX 是典型的高 beta 基础设施资产，对宏观流动性非常敏感。

主要传导路径：

- 利率与美元走强：
  会压制这类依赖远期增长与风险偏好的 alt-L1 资产。
- 风险偏好回升：
  会提升资金对平台币、基础设施币和高 beta 公链的配置意愿。
- BTC dominance 上升：
  通常会压制 AVAX 这类中大型 alt 的相对表现。
- 稳定币扩张与链上金融主题升温：
  会强化 Avalanche 的机构/RWA 叙事，尤其当市场开始重视 tokenization 基础设施时。

所以 AVAX 的宏观属性并非避险；核心在于：

- 平时更像高 beta 成长型平台资产；
- 当“数字资产基础设施”与“风险偏好修复”叠加时，会获得额外弹性。

## 12. 经济政策的影响

AVAX 的政策敏感度，主要来自五条线：

- 稳定币政策
  Avalanche 正在明显押注稳定币和机构结算，稳定币立法越清晰，越利好其平台叙事。
- 证券代币化政策
  Progmat 这样的案例，本质上高度依赖证券和数字资产监管框架的可操作性。
- 跨境合规
  Avalanche 若想吃到更多亚洲和全球机构流量，跨境托管、KYC、发行和转让规则都会影响采用速度。
- 交易所与资产分类
  AVAX 作为老牌 L1，监管定位通常比 meme 或高争议治理币更稳定，但这并不意味着完全无风险。
- 协议内部“经济政策”
  Etna、ACP、L1 费用机制、激励分发等，实际上也是平台内部经济政策，直接决定 AVAX 捕获强弱。

我的判断：

- 外部政策上，AVAX 并非最脆弱的币，反而可能是稳定币和代币化证券政策改善的受益者；
- 内部政策上，Avalanche 更关键的是如何让 L1 平台扩张与 AVAX 经济更紧密挂钩。

## 13. 股市的影响

AVAX 与股市的关系，更像：

- 相关股市风格：
  `高贝塔科技股 / 云基础设施 / 金融科技基础设施 / 风险资产`
- 关键传导路径：
  流动性、风险偏好、成长股估值扩张收缩、crypto proxy equities 的风险偏好传导、以及市场对 tokenization/fintech infra 的再定价
- 最相关的股票或指数代理：
  `Nasdaq 100`、高成长软件与云基础设施板块、以及 `COIN`、`HOOD` 这类加密风险偏好代理股
- 这种影响是短期情绪主导，还是中期估值锚定：
  短期更偏情绪；中期则会通过“AVAX 是否数字资产基础设施资产”来影响估值倍数

更直白地说：

- 当美股成长股、AI/软件基础设施和 crypto proxy 股票一起走强时，AVAX 往往更容易获得弹性；
- 但如果市场进入高利率、低估值环境，AVAX 通常会比 BTC 这类核心资产更脆弱。

## 14. 国际战争的影响

国际战争对 AVAX 的影响，短期和长期并不一致。

- 第一阶段，短期偏利空：
  战争升级、油价上行、美元走强、全球风险偏好下降时，AVAX 这类高 beta alt 往往先被减仓。
- 第二阶段，中长期可能有结构性利好：
  如果全球金融体系更加碎片化，合规跨境结算、权限型链、数字证券基础设施和更可控的区块链环境，反而可能得到更多需求。

具体传导路径：

- 风险偏好压缩：
  直接打击 AVAX 价格。
- 制裁与资本管制：
  可能推动更多地区探索合规数字资产基础设施。
- 网络安全与数字主权：
  可能提高对可定制链和机构级链上基础设施的需求。
- 大宗商品和能源冲击：
  通过宏观紧缩和流动性恶化，间接压制 AVAX。

所以更准确的结论是：

- 战争对 AVAX 的短期价格通常偏负面；
- 但对 Avalanche 作为“可配置金融基础设施”的长期叙事，未必完全是坏事。

## 16. 智能合约开发用什么语言

如果说的是 Avalanche 主生态，尤其是 C-Chain 或 Subnet-EVM / EVM 型 Avalanche L1：

- 主要智能合约语言：
  Solidity
- 常见兼容语言：
  Vyper
- 执行环境：
  EVM
- 如果构建自定义 Avalanche L1：
  可根据所选 VM 自定义开发栈，不必局限于 EVM

需要强调的是：

- 官方 Builder Hub 的主流合约部署路径是 `Solidity + EVM`；
- 文档明确说明 Avalanche C-Chain 与 Subnet-EVM 当前支持 `Cancun` EVM 版本；
- Avalanche 的真正特殊性不在合约语言本身，而在于你可以选择发一条自己的 L1，并决定是否继续使用 EVM。

## 关键风险

- 平台扩张发生了，但价值主要停留在各条 L1 和应用层，没有明显回到 AVAX。
- RWA 与机构案例很多，但多数仍停留在试点和展示层。
- C-Chain base fee 降低虽有利采用，但会压缩单位交易收入。
- AVAX 的叙事过于复杂，市场不愿意为复杂故事支付高估值。
- 与 SOL、SUI 等竞争时，Avalanche 缺少最强的单点爆发心智。
- 宏观逆风下，AVAX 作为高 beta alt-L1 会率先承压。

## 未证实但值得跟踪的问题

- `Retro9000` 到底会不会显著提高 AVAX burn，而非只是阶段性激励效果。
- Progmat 专用 Avalanche L1 的真实资产增长和交互频率会有多高。
- FUSD 能否真正带来亚洲机构资金与全球链上流动性的桥接。
- Avalanche L1 的数量增长，是否会同步带来 ICM 的广泛采用和跨链消息量上升。
- 未来是否会出现一条真正出圈的 Avalanche L1，形成类似“Off The Grid”那样更强的应用牵引。
- AVAX 是否能逐步形成比现在更稳定的“平台股权代理”认知。
