## 与上次相比的关键变化

本次为首次覆盖，不存在内部版本对照。若只看最近 9-12 个月，CKB 最值得关注的变化有六个：

- 2025-07-01，CKB Edition Meepo 硬分叉在主网上线，核心变化是 `CKB-VM V2` 与 `Spawn`，这让跨脚本调用和更复杂的链上程序设计变得更现实，开发者工具链也因此进入新阶段。
- 2026-01-21 的官方 Dev Log 确认：`Quantum Resistant Lock Script` 已部署到 CKB 主网，同时轻客户端开始针对移动端原生支持与 SQLite 后端优化。这代表 CKB 仍在围绕 `长期安全性 + 低资源接入` 两条线推进，而不是停留在旧叙事。
- 2026-02-05 的 Dev Log 显示：CKB 已新增 `OpenRPC JSON 规范与交互式浏览器`、`日志订阅 API`、RocksDB 架构重构，以及面向 AI 编程助手更友好的智能合约脚手架，说明团队把更多精力放在开发体验、可观测性和基础设施可维护性上。
- 2026-03-17，主节点发布 `v0.205.0`，带来 CKB-TUI 的 Terminal 模块、代理协议支持、导入导出能力增强、日志订阅与安全修复。短期看这不是用户爆发式增长事件，但对节点运营和工具完善是实质进展。
- Fiber Network 这条支付与交换通道线没有停。GitHub 发布记录显示 `v0.7.0` 于 2026-02-12 发布，新增单向通道、trampoline routing、路径查找优化等；`v0.7.1` 又在 2026-02 下旬继续修补支付与通道打开可观测性。CKB 的“Bitcoin 同构扩展层”叙事仍在推进，而不是被放弃。
- 市场层面，CKB 现在的矛盾非常清楚：技术和路线图在持续推进，但价格、市值、交易深度和链上活跃度仍停留在小市值边缘层，说明市场还没有把这些进展转换成明确估值溢价。

## 解锁与供给压力快照

CKB 当前最重要的供给事实不是“VC cliff 解锁”，而是 `程序化发行`。

- 最近一次重大供给事件：第一次减半已在 2023-11-19 完成，基础发行的区块奖励进入减半后的区间。
- 下一次已知重大供给事件：第二次减半预计在 `2027-11-20 09:24:40` 左右发生。按 2026-03-19 的浏览器倒计时，距离下一次减半约 `611 天`。
- 当前主要供给来源一：`Base Issuance`。减半后的基础发行约为每天 `575.34 万 CKB`。
- 当前主要供给来源二：`Secondary Issuance`。官方文档给出的固定年化二级发行是 `13.44 亿 CKB`，折合每天约 `368.22 万 CKB`。
- 按官方机制估算的未来 gross 发行：
	- 未来 30 天：约 `2.8307 亿 CKB`
	- 未来 90 天：约 `8.4921 亿 CKB`
	- 未来 365 天：约 `34.44 亿 CKB`
- 按 MiningPoolStats 最近 24 小时显示的实际 `Emission (24h)` 估算，当前更接近净新增流通的节奏约为 `615.26 万 CKB / 天`：
	- 未来 30 天净年化参考：约 `1.8458 亿 CKB`
	- 未来 90 天净年化参考：约 `5.5374 亿 CKB`
	- 未来 365 天净年化参考：约 `22.46 亿 CKB`

受益方与供给分配机制：

- 基础发行主要支付给矿工。
- 二级发行按网络状态分配给矿工、Nervos DAO 存款者，以及 Treasury。
- 这不是团队或投资人线性解锁，而是 `programmatic issuance`。

为什么这很重要：

- 对 CKB 而言，未来 12 个月最现实的抛压来源是 `矿工卖压 + 通胀稀释`，不是突然到期的私募解锁。
- 这对持币人有两个含义：
	1. 供给路径更可预测；
	2. 如果链上使用、Nervos DAO 存款和状态占用增长不明显，通胀压力就更容易体现在价格上。

结论：

- CKB 的供给压力是 `可预测但持续的`；
- 好处是没有典型 altcoin 那种大额 cliff 爆雷；
- 坏处是如果需求端不改善，持续性程序化发行会长期压制估值中枢。

## 节点与验证者概况

CKB 是 PoW 链，不存在 PoS 式“验证者集合”，因此这里必须分开看：`验证者` 不适用，但 `矿工 / 矿池 / 节点` 很重要。

- 验证者数：`不适用`。CKB 使用 PoW 共识，不是验证者质押网络。
- 官方公开的实时全节点总数：目前缺乏一个像 Cosmos/Solana 那样标准化、持续更新且官方确认的总节点数口径，因此不能编造一个节点数。
- 当前网络哈希率：约 `115.82 PH/s`。
- 当前难度：约 `1.3664E`。
- 根据 MiningPoolStats，当前主要矿池哈希率大致为：
	- F2Pool：`44.87 PH/s`
	- ViaBTC：`27.27 PH/s`
	- 2miners：`12.37 PH/s`
	- dogpool.work：`11.46 PH/s`
	- ntminerpool.com：`9.32 PH/s`
	- Antpool：`8.37 PH/s`
- 最近 1000 个区块分布显示：
	- 前 2 名出块方合计约 `73.0%`
	- 前 3 名合计约 `84.4%`
	- 前 5 名合计约 `98.3%`

这意味着什么：

- 好消息是，CKB 作为一个小市值 PoW 链，当前哈希率并不低，短期安全预算并非脆弱到“轻易可被拿下”的程度。
- 风险在于，`出块权高度集中于少数矿池` 是当前最清晰的中心化问题。即使矿池背后可能分散了很多独立矿工，矿池层面的协调与审查风险依然存在。
- 基金会 2025-06-04 的帖文强调 CKB 由广泛的用户、矿工和 full nodes 运行，网络无法被单方面关闭；这个判断在原则上成立，但若要论证 `去中心化质量很强`，当前最欠缺的是可验证、持续更新的节点分布与地域数据。

结论：

- `安全性` 目前尚可；
- `去中心化质量` 不能简单给高分，尤其要盯住矿池集中度；
- CKB 如果未来想被市场重新当作严肃的 Bitcoin 基础设施资产定价，节点与矿池分布透明度需要更强。

## 赛道定位

- 主赛道：Bitcoin 扩展基础设施 / UTXO 智能合约底层
- 次赛道：L1 公链 / 支付与通道网络 / 资产发行与存储层

把 CKB 简单归类为“普通 L1 公链”会低估它真正想做的事。

更准确的框架是：

- CKB 本体是一条 PoW + UTXO + RISC-V 的基础链；
- 它想承接的是 `Bitcoin 资产、Bitcoin 兼容执行、链下支付通道、跨资产交换` 这些需求；
- RGB++、Fiber、Nervos DAO、状态租金模型，都是围绕这一定位搭建的。

所以评估 CKB，最合理的方式不是问“它能不能像 SOL 那样卷活跃地址”，而是问：

1. 它能不能在 Bitcoin 可编程化这条路里占到一席之地；
2. 即便链和工具链做出来，`CKB 代币本身` 能否抓到足够价值。

## 核心投资逻辑

CKB 最简洁的多头逻辑是：

- 它是少数把 `PoW 安全性`、`UTXO 模型`、`RISC-V 通用执行`、`状态租金` 和 `Bitcoin 扩展` 放在同一套设计里的资产；
- 如果未来 1-3 年 Bitcoin 生态继续从“单纯价值存储”向“资产发行、支付通道、链下交换、可编程资产”延伸，CKB 有机会成为其中的底层承载层，而不是纯粹旁观者；
- 与很多 altcoin 不同，CKB 当前已接近全流通，估值受 VC 解锁扰动较小，小市值本身保留了较大的估值弹性；
- 下一轮真正能让 CKB 走出长期低估的，不会是单独一个技术发布，而是 `Fiber / RGB++ / 存储占用 / DAO 锁仓 / 费用与矿工经济` 形成某种可验证闭环。

一句话概括：

CKB 的投资逻辑不是“又一条公链补涨”，而是 `Bitcoin 扩展基础设施如果真的成形，CKB 可能是被低估的底层受益者`。

## 反方观点与证伪条件

最强的反方逻辑同样很坚实：

- 第一，CKB 的产品定位一直偏“工程师友好”，但真实用户规模、资金规模和主流叙事抓手都弱，这让它非常容易沦为“技术上很完整，市场上很边缘”的项目。
- 第二，RGB++、Fiber、状态租金这些概念都很有辨识度，但真实需求可能不足。Bitcoin 生态的绝大部分流量未必需要这套复杂堆栈。
- 第三，链变强不等于币变强。就算 CKB 成为某种后台基础设施，用户也可能只在用 BTC、稳定币和 RGB++ 资产，而不是主动持有大量 CKB。
- 第四，PoW 带来的是清晰的安全模型，但也带来持续矿工卖压与矿池集中问题。对一个 7000 多万美元市值资产而言，这个问题不能忽略。

最关键的证伪条件：

- 如果未来 6-12 个月，Fiber、RGB++、轻客户端、开发工具继续更新，但链上交易、存储占用、Nervos DAO 锁仓、矿工经济与市场流动性都没有明显改善，那么“技术进展会转化为估值提升”的逻辑将被明显削弱。
- 如果 Bitcoin 可编程化的主要流量继续流向 Stacks、Rootstock、Core、Lightning 生态或中心化托管层，而 CKB 仍缺少真正破圈的资产和应用，CKB 很可能继续处于估值折价状态。
- 如果矿池集中度进一步抬升，而总安全预算又无法随价格同步改善，市场会对其长期安全性打折。

## 市场可能忽略的变量

- 市场可能低估的变量一：`几乎全流通` 本身就是一种稀缺属性。很多中小市值公链表面便宜，实际上前方全是解锁；CKB 没有这个典型雷。
- 市场可能低估的变量二：`状态租金` 不是营销语，而是一套试图把“链上状态占用”与“代币资本成本”绑定起来的设计。若 RGB++ 资产和更复杂的 Cell 使用变多，CKB 的需求基础可能比传统 gas token 更稳定。
- 市场可能低估的变量三：`Nervos DAO` 让持币者有了对冲二级发行稀释的工具，这使得 CKB 的持有结构与普通通胀币不同。
- 市场可能低估的变量四：CKB 当前与其说在赌“公链大战”，不如说在赌 `Bitcoin 原生与比特币相关资产是否需要一条更灵活的底层`。如果这个方向对了，CKB 的可比组会被重新定义。
- 市场可能低估的变量五：最新 Dev Log 已经出现 `链上治理` 与 `Nervos DAO treasury` 重新设计的线索。如果二级发行分配逻辑优化，代币价值捕获可能改善。

## 对标项目比较

CKB 当前最合适的对标组并不完美。我选 `Stacks`、`Rootstock`、`Core` 作为近似可比组，因为它们都试图吃到 Bitcoin 资产、Bitcoin 可编程化或 Bitcoin 安全叙事带来的增量。

- 对标对象：Stacks、Rootstock、Core
- 为什么选它们做对比：
	- `Stacks` 代表更成熟的 Bitcoin 应用与智能合约叙事；
	- `Rootstock` 代表 EVM 路线的 Bitcoin 侧扩展；
	- `Core` 代表更偏市场化和高 beta 的 Bitcoin 相关基础设施代币。
- 相对优势：
	- CKB 的 `PoW + UTXO + RISC-V + 状态租金` 组合更底层、更统一，也更容易讲“长期 Bitcoin 基础设施”而不是单一应用链故事；
	- 供应几乎全流通，结构上比许多同类项目更干净；
	- Fiber 与 RGB++ 的路线更强调多资产与支付通道，而不只是把 EVM 再复制一遍。
- 相对劣势：
	- 用户侧和资本市场侧的认知都更弱；
	- 缺少像 Stacks 那样已经较有存在感的 BTCFi 生态标签；
	- CKB 的开发者友好性提升很多，但主流应用破圈仍不明显；
	- 现阶段估值便宜有其合理性，因为真实采用和费用还不够强。
- 当前估值相对同类是溢价、平价还是折价：
	- 我认为 CKB 目前处于 `有基本面折价理由的深度折价`。也就是说，市场没有白白错杀它，而是在给“采用尚未证明”打折；但如果 Fiber / RGB++ 真的拿到数据，重估空间会很大。

## 执行摘要

CKB 当前最重要的判断，不是它是不是“老公链”，而是它能否从 Bitcoin 扩展基础设施重新获得定价权。过去 9 个月里，CKB 的底层更新并不弱：Meepo 硬分叉主网落地、量子抗性锁脚本上线、移动轻客户端和 SQLite 后端推进、OpenRPC 与日志订阅完善、RocksDB 架构优化、以及 Fiber 持续发布到 v0.7.1。问题在于，这些进展暂时更多体现在开发者和节点运营层，而不是用户增长、交易量和收入上。链的发展前景我给 `中性偏强`，因为它在 Bitcoin 相关基础设施里的路线具有明显差异化；但代币投资前景我只给 `中性`，因为价值捕获目前仍偏弱，矿工卖压和低流动性也是真实存在的。未来 12 个月，CKB 最关键的验证点有三件事：`Fiber / RGB++ 是否出现看得见的使用数据`、`Nervos DAO 与状态租金模型是否能持续吸收稀释压力`、`链上治理和 Treasury 设计是否改善价值回流逻辑`。如果三者中至少两条成立，CKB 会从“技术存在感高、市场存在感低”的项目，变成真正具备重估条件的 Bitcoin infra 小市值资产；如果不成立，它大概率继续维持长期折价。

- 2025-07-01，`CKB Edition Meepo` 主网硬分叉上线。最重要的新能力是 `Spawn` 与 `CKB-VM V2`，这明显提高了复杂脚本与更丰富链上逻辑的可行性。

	- 面向 RISC-V 64 的 `bn128` 库已发布。

这些变化的重要性在于：

- CKB 的研发并没有停；
- 它正在从“老牌公链”转成“Bitcoin 相关底层 + 通道网络 + 资产层”；
- 但生态层最缺的仍是明确的 `用户、资产、费用` 数据，而不是代码提交本身。

## 2. 最近 X 上的讨论

CKB 在 X 上的讨论有一个很鲜明的特征：`开发更新比市场热度强，GitHub 活跃度比社媒声量更领先`。

- 2026-03-04，`@RunningCKB` 发帖总结 2025 年度进展，重点提到 `Meepo hard fork` 和 `first quantum-resistant wallet`。这类发帖传递的核心信息是：团队希望市场把 CKB 视作长期安全与基础设施型项目，而不是短线叙事币。
- `@FiberDevs` 在此前的主网与 Dev Log 帖文里，最重要的讨论点不是“多快多便宜”，而是 `Fiber is not a rollup, not a sidechain, but a channel network`。这说明团队在刻意把 Fiber 定位为更接近 Lightning 的支付层，而非另一条普通扩容链。

- 对投资者来说，这是机会也是风险：机会在于低关注度，风险在于如果 6-12 个月后仍只剩开发者自说自话，估值就难以扩张。

## 3. 经济模型

CKB 的经济模型是整个项目最独特、也最容易被忽视的部分。

### 3.1 CKByte 的本质

官方文档把 CKB 描述为一种与链上存储直接绑定的原生资产：`1 CKByte 约等于 1 byte 的状态存储权`。这和一般公链把代币主要当 gas 不同。

这带来三层费用或价值锚：

- `Cycles`：智能合约计算资源；
- `Transaction Fees`：区块安全与打包费用；
- `State Rent`：长期存储占用的隐性资本成本。

### 3.2 发行结构

- `Base Issuance`：总量上限 `336 亿 CKB`，每四年减半一次，已经完成第一次减半。
- `Secondary Issuance`：固定每年 `13.44 亿 CKB`，用来维持矿工激励和状态存储成本的长期可持续性。

### 3.3 二级发行如何分配

二级发行并不是简单给矿工，而是按网络状态分配给：

- 矿工
- Nervos DAO 存款者
- Treasury

官方文档的关键点在于：

- 存储链上状态的人，实际上要承担通胀带来的隐性折旧；
- 不占用状态、而把 CKB 存进 Nervos DAO 的人，可以对冲这种稀释。

### 3.4 Nervos DAO 的意义

Nervos DAO 是 CKB 模型里非常重要但经常被忽视的装置。

- 对普通持币者来说，它提供了一个对冲二级发行稀释的路径；
- 对网络来说，它鼓励不活跃的状态被清理，避免无效状态长期堆积；
- 对投资视角来说，它让 `持有 CKB` 与 `无脑裸持一个通胀币` 不完全一样。

### 3.5 经济模型的优点

- 与链上状态绑定，理论上比纯 gas 资产更有长期锚定逻辑；
- 供应路径透明，没有依赖治理随意调参的通胀黑箱；
- DAO 机制使得持有者不是被动挨稀释；
- 近全流通结构让估值分析更干净。

### 3.6 经济模型的现实问题

- 当前链上使用量仍偏低，导致 `状态租金模型的长期优雅` 还没有变成 `短中期估值支撑`；
- 二级发行仍会转化成持续卖压，尤其当矿工和非长期持有者选择兑现；
- 如果 RGB++ / Fiber 的流量主要沉淀在 BTC、稳定币或上层资产，CKB 代币未必能充分受益。

结论：

CKB 的 tokenomics 设计是 `有想法且自洽的`，比绝大多数中小公链更值得研究；但它当前最大的问题不是设计错，而是 `需求强度还不够把设计优势兑现成价格优势`。

## 4. 前景分析

如果单看链的发展前景，我对 CKB 的判断是 `中性偏强`；如果看代币的投资前景，我给 `中性`。

### 4.1 为什么链的发展前景不是弱

- CKB 不是一个“已经放弃讲新故事”的旧项目。最近 12 个月，主网硬分叉、量子抗性、轻客户端移动化、数据库优化、zkVM 验证研究、Fiber 版本推进，说明它仍在推进完整路线。
- 在 Bitcoin 扩展赛道里，CKB 的差异化非常明确：`PoW`、`UTXO`、`RISC-V`、`状态租金`、`Fiber 通道网络`、`RGB++ 资产扩展`。这不是抄现成的 EVM 公链模板。
- 如果未来市场开始更认真地区分“谁是真正在做 Bitcoin 资产与支付基础设施，谁只是蹭 Bitcoin beta”，CKB 会重新进入可比较名单。

### 4.2 为什么代币前景不能直接给强

- CKB 现在最缺的不是技术路线，而是 `代币价值回流证据`。
- 链的开发进展很多，但费用、交易数、生态资产规模、存储占用增长，还没有展示出足以支撑高估值重估的强度。
- 程序化发行和矿工卖压是持续存在的现实约束。

### 4.3 未来 12-24 个月的核心问题

不是“CKB 还能不能继续更新”，而是：

1. RGB++ 与 Fiber 会不会形成真实使用；
2. 这些使用会不会增加 CKB 容量占用、DAO 锁仓、费用与流动性；
3. 市场会不会重新把它从“老 alt”切换为“Bitcoin infra”。

结论：

- 链的前景：`中性偏强`；
- 币的前景：`中性`；
- 想把币的评级抬高，必须先看到使用与价值捕获证据，而不是再听一轮新叙事。

## 5. 捕获价值分析

CKB 的价值捕获路径和普通公链差异很大，必须拆开看。

### 5.1 链上活动如何可能转化为 CKB 价值

- 第一条路径：`状态占用`。资产、数据、Cell 需要占用 CKB 容量，理论上越多状态被长期使用，对 CKB 的资本占用越强。
- 第二条路径：`交易与计算`。链上执行消耗 Cycles 和交易费，部分构成矿工收益和安全预算。
- 第三条路径：`Nervos DAO`。持有者为了对冲稀释，会把空闲 CKB 存入 DAO，这会减少可流通抛压。
- 第四条路径：`Fiber / RGB++ 的底层锚定`。如果更多 BTC 相关资产和支付通道必须依赖 CKB 作为结算或存储底层，CKB 的需求会更接近“底层资源”而不是单纯投机工具。

### 5.2 当前价值捕获的问题

- 目前最核心的问题是，`这四条路径都还没有强到让市场愿意给高倍数`。
- 存储占用逻辑长期成立，但短期仍然难以替代真实费用和现金流类指标。
- 程序化发行相当可观，意味着价值捕获必须跑得比稀释更快，币价才容易持续上行。
- Fiber 和 RGB++ 成功的话，价值也可能先流向使用其网络的资产与支付场景，而非立刻反映到 CKB 本身。

### 5.3 我的结论

CKB 的价值捕获设计比很多公链 `更严肃、更耐久`，但当前价值捕获强度仍偏弱。

更直接地说：

- CKB 不是没有价值捕获逻辑；
- 它的问题在于 `逻辑已存在，数据仍不足`。

## 9. 与 AI 相关的重点分析

CKB 与 AI 的关系目前属于 `开发工具层相关，产品需求层弱相关`。

更具体地说：

- 不是 AI 基础设施链：CKB 不是去中心化算力网络，不是模型推理网络，也不是数据标注网络。
- 不是 AI 应用链：目前没有证据显示 AI 工作负载是 CKB 主网需求的核心来源。
- 有一定 AI 开发工具相关性：2026-02-05 Dev Log 提到团队发布了更适合 AI coding assistants 使用的智能合约 boilerplate，这会让 CKB 更容易被 AI 辅助开发流程接入。
- 潜在间接相关性：若未来 AI agent 需要管理 UTXO 资产、跨链支付或可验证执行环境，CKB 的架构理论上可承接部分需求，但这离当前估值主线仍很远。

结论：

- AI 对 CKB 目前是 `辅助项`，不是核心估值驱动；
- 如果市场把 CKB 当 AI 币来交易，我会认为那是错误定价；
- 真正该盯的是 Bitcoin 扩展与支付基础设施，而不是 AI 叙事。

## 11. 宏观经济对此标的的影响

CKB 对宏观的敏感度，本质上是 `小市值高 beta 风险资产`。

- 当全球流动性改善、降息预期升温、美元走弱、BTC 强势且市场愿意向 Beta 外溢时，CKB 往往受益。
- 当市场重新聚焦“只有 BTC 值得买”时，CKB 这类 Bitcoin infra 标的反而容易被边缘化，因为资金先买最确定性的 BTC 本身。
- 如果 BTC ETF 继续吸收大额资金，而资金不向 Bitcoin infra 外溢，CKB 只能享受弱相关正面情绪，难以拿到独立估值提升。
- 如果宏观进入再通胀、能源价格高企或风险偏好恶化阶段，CKB 的 PoW 属性和小市值属性都会放大波动。

结论：

- 宏观流动性改善对 CKB 是必要但不充分条件；
- CKB 真正需要的是 `宏观风险偏好改善 + Bitcoin infra 再定价` 两者共振。

## 12. 经济政策的影响

政策对 CKB 的影响，主要通过四条路径传导：

- 第一，`交易所可得性`。小市值资产高度依赖中心化交易所流动性，若交易平台对中小币种进一步收缩支持，CKB 估值会受压。
- 第二，`Bitcoin 相关政策`。如果主要司法辖区对 Bitcoin 二层、资产发行、稳定币和链下支付环境更友好，CKB 所在赛道的空间会被放大。
- 第三，`挖矿与 PoW 政策`。PoW 若在部分区域遭遇更严格的能耗或合规约束，会影响矿工结构与安全预算。
- 第四，`稳定币与跨境支付政策`。Fiber 如果未来真要承接多资产支付和交换，稳定币监管环境会直接影响其商业化可能性。

我的判断：

- 短期政策对 CKB 的影响更多体现在 `交易所流动性` 和 `市场可访问性`；
- 中期政策影响则取决于 Bitcoin 相关资产与支付层在监管上是否获得更清晰位置。

## 13. 股市的影响

CKB 不是一个会直接被美股财报驱动的资产，但它仍受股市风险偏好影响。

- 相关股市风格：更接近 `高贝塔风险资产 / 加密概念外溢资产`，而不是纯 AI 科技股代理。
- 关键传导路径：
	- 纳斯达克与成长股风险偏好改善，会提升市场对中小市值加密资产的容忍度；
	- 如果 `COIN`、`MSTR`、`MARA`、`RIOT`、`CLSK` 这类加密和比特币相关股票走强，市场更容易重新关注 Bitcoin 周边基础设施；
	- ETF 和基金的风险预算扩张，也会间接利好 CKB 这类边缘资产。
- 最相关的股票或指数代理：`NASDAQ 100`、`COIN`、`MSTR`、`MARA`、`RIOT`、`CLSK`。
- 这种影响是短期情绪主导，还是中期估值锚定：
	- 短期更多是情绪和流动性驱动；
	- 中期只有当 Bitcoin 基础设施赛道整体被资本市场重新认识时，才会形成更稳固的估值锚。

## 14. 国际战争的影响

国际战争与地缘冲突对 CKB 的影响是 `偏间接、偏混合`。

- 关键传导路径一：`能源价格冲击`。CKB 是 PoW 资产，若战争推高能源价格，会抬升矿工成本，对安全预算和卖压都有影响。
- 关键传导路径二：`避险资金先流向 BTC`。如果地缘冲突强化 BTC 作为避险或中立资产的吸引力，CKB 可能获得弱外溢，但通常先受益的是 BTC 本身，而非 CKB。
- 关键传导路径三：`跨境支付与制裁环境`。如果全球支付碎片化加剧，像 Fiber 这种支付与交换网络的潜在叙事会更有吸引力，但这属于中长期结构变量，不是短期价格驱动。
- 关键传导路径四：`网络安全与基础设施风险`。战争期间网络攻击与基础设施不稳定风险上升，矿池和节点服务也会受到更大考验。

这个资产更可能受益、受损，还是影响中性：

- 短期偏 `中性偏负面`，因为风险偏好下降和能源成本上升通常先压制小币；
- 中长期存在少量结构性受益可能，前提是跨境支付与中立结算需求真的上升到能带动 Bitcoin 相关基础设施。

## 16. 智能合约开发用什么语言

CKB 没有类似 Solidity 那样唯一主导的“官方智能合约语言”。更准确的说法是：

- 执行环境：`CKB-VM`，底层基于 `RISC-V`。
- 官方当前最推荐的脚本开发语言：`Rust`。
- 也支持 `C` 直接编译到 RISC-V。
- 还支持通过解释器在 CKB-VM 中运行 `JavaScript` 和 `Lua`。
- 文档明确表示，理论上只要工具链合适，几乎任何语言都可以运行在 CKB 上。
- JavaScript / TypeScript 侧，官方当前推荐的开发 SDK 是 `CCC (Common Chain Connector)`。

因此，如果用一句话总结：

`CKB 的核心不是一门合约语言，而是一个基于 RISC-V 的通用执行环境；实际开发中 Rust 最值得优先考虑。`

## Key Risks

- 采用风险：Fiber、RGB++ 和 Bitcoin 扩展叙事可能迟迟无法形成真实用户与资产沉淀。
- 价值捕获风险：链在变强，但 CKB 代币不一定同步受益。
- 持续供给风险：程序化发行与矿工卖压会持续存在。
- 矿池集中风险：近期出块权分布偏集中，安全性与审查阻力需要持续观察。
- 流动性风险：市值与成交额较低，容易放大上下波动。
- 竞争风险：Stacks、Rootstock、Core 以及其他 Bitcoin 生态方案都在争夺同一叙事空间。

## 未证实但值得跟踪的问题

- Fiber 主网当前的真实活跃节点数、通道数、支付成功率和交易规模，到底有多大。
- RGB++ 资产和跨资产交换是否会在未来 12 个月显著扩张。
- Nervos DAO Treasury 与二级发行分配方案是否会出现实质性优化。
- 下一次硬分叉的具体范围、时间窗口和会否带来更强的价值回流设计。
- 官方是否会提供更标准化、持续更新的全节点分布与地域透明度数据。
